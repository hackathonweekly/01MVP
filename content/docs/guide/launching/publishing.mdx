## 发布前核心准备

无论你选择哪个市场，以下准备工作都是必要的:

### 产品素材准备清单

- [ ] **产品标识**:
  - 高质量logo (多种尺寸)
  - 一句话产品定位
  - 简洁有力的产品描述(长短两版)

- [ ] **视觉资料**:
  - 5-8张高质量产品截图
  - 30-60秒产品介绍视频/GIF
  - 品牌配色和设计元素

- [ ] **文档资料**:
  - 用户指南/快速入门
  - 常见问题(FAQ)
  - 基本隐私政策和用户协议

### 技术准备

- [ ] 确保产品稳定，核心功能正常运行
- [ ] 设置错误监控和用户反馈渠道
- [ ] 准备版本更新和迭代机制

### 发布时机选择

选择合适的发布时间可以提高初始曝光:

- **避开重大节假日**: 媒体和用户注意力分散
- **避开科技巨头发布会**: 如苹果WWDC、谷歌I/O等
- **工作日上午/中午**: 通常用户活跃度高
- **考虑时区差异**: 针对国际市场时考虑目标地区的黄金时段

## 发布后行动计划

产品发布只是开始，请确保有以下后续计划:

1. **24-48小时响应计划**: 
   - 指派团队成员密切监控用户反馈
   - 准备快速修复关键问题
   - 主动与早期用户互动

2. **数据收集与分析**:
   - 追踪注册/下载量、激活率、留存率
   - 识别用户流失的关键环节
   - 记录并分类用户反馈

3. **快速迭代计划**:
   - 基于初始反馈规划1-2周内的首次更新
   - 优先解决影响用户体验的关键问题
   - 向用户通报改进计划

产品成功发布后，下一步是执行系统化的[用户获取策略](/docs/launching/user-acquisition)，获得你的前100个真实用户。
