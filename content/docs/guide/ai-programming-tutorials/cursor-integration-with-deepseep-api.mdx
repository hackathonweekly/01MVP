> 💡 **MVP成本优化**: 使用DeepSeek API替代Cursor内置AI可大幅降低开发成本，同时保持高质量代码生成能力，让你的MVP研发预算更合理。

## 为什么选择DeepSeek API？

DeepSeek V3是国内领先的AI模型，具有以下优势：

- **卓越代码能力**: 代码生成与理解能力与Claude Sonnet相当
- **极具性价比**: 成本仅为OpenAI/Claude的10-20%
- **高效开发**: 支持快速迭代，加速MVP开发周期
- **中文优化**: 对中文场景和需求有更好的理解

### 价格优势对比

| 模型 | 输入成本 (百万tokens) | 输出成本 (百万tokens) | 相对价格 |
|------|---------------------|---------------------|---------|
| **DeepSeek V3** | ¥0.1-1 | ¥2 | 基准 (1x) |
| GPT-4 | ¥60 | ¥120 | 约60x |
| Claude 3 Sonnet | ¥24 | ¥82 | 约40x |

> ⚠️ **注意**: 接入自定义API后，Cursor只能使用Chat模式，无法使用Compose/Agent模式，但对于大多数MVP开发场景已经足够。

## 第一步：获取DeepSeek API密钥

### 方法一：直接从DeepSeek官网获取

1. 访问[DeepSeek平台](https://platform.deepseek.com/usage)
2. 注册并登录账号
3. 在API Keys页面创建新的API密钥
4. 复制并安全保存API密钥

![DeepSeek官网API页面](/images/cursor-integration-with-deepseep-api.001.png)

### 方法二：通过Sealos AI Proxy获取（推荐）

使用[Sealos](https://sealos.run)的好处：
- 一个API密钥访问多家国内大模型
- 价格与官方完全一致，零加价
- 简化账户管理，无需多平台注册

操作步骤：
1. 注册并登录[Sealos平台](https://cloud.sealos.run)
2. 打开主页的"AI Proxy"应用
3. 点击"API Keys"创建新密钥
4. 复制并保存生成的API密钥

| ![Sealos主页](/images/cursor-integration-with-deepseep-api.002.png) | ![API密钥创建](/images/cursor-integration-with-deepseep-api.003.png) |
|:---:|:---:|

## 第二步：在Cursor中配置DeepSeek API

1. 打开Cursor应用
2. 点击顶部菜单栏的`Cursor > Settings > Cursor Settings`
3. 在设置面板中，选择`Models`选项卡

![Cursor设置入口](/images/cursor-integration-with-deepseep-api.004.png)

4. **重要**: 取消勾选所有其他已激活模型
5. 点击"Add Model"添加自定义模型
6. 在模型名称处输入`deepseek-chat`并回车

![添加自定义模型](/images/cursor-integration-with-deepseep-api.005.png)

7. 设置API基础URL:
   - 如使用DeepSeek官方API: `https://api.deepseek.com/v1`
   - 如使用Sealos AI Proxy: `https://aiproxy.gzg.sealos.run/v1`
   
   > ⚠️ **注意**: URL末尾的`/v1`不能省略

8. 输入之前获取的API密钥，点击"Verify"验证
9. 验证成功后保存设置

![配置API URL和密钥](/images/cursor-integration-with-deepseep-api.006.png)

## 第三步：开始使用DeepSeek驱动的Cursor

1. 使用快捷键`Cmd+L`(Mac)或`Ctrl+L`(Windows)打开Cursor聊天窗口
2. 输入编程相关问题或需求
3. 体验DeepSeek V3模型的高效代码生成能力

> 💡 **提示**: DeepSeek-chat 接口已全面升级为 DeepSeek-V3，无需特殊设置即可使用最新模型。

## 常见问题与解决方案

### API连接失败

**可能原因**:
- API URL格式错误
- API密钥无效或过期
- 网络连接问题

**解决方案**:
1. 确认URL末尾包含`/v1`
2. 重新生成并复制完整的API密钥
3. 检查网络连接，必要时使用VPN

### 模型响应质量不理想

**可能原因**:
- 提示词不够具体或清晰
- 上下文信息不足

**解决方案**:
1. 使用更详细的提示词，明确说明需求
2. 提供相关代码示例或背景信息
3. 分步骤提问，逐步引导模型

### 计费与余额问题

**查看用量**:
- DeepSeek官网: [https://platform.deepseek.com/usage](https://platform.deepseek.com/usage)
- Sealos平台: AI Proxy应用中的"用量"选项卡

**充值方式**:
- DeepSeek官网支持支付宝、微信支付
- Sealos平台支持多种支付方式，最低充值额度更低

## 高级使用技巧

### 优化输入成本

DeepSeek V3模型的输入成本分为两档:
- 缓存命中: ¥0.1/百万tokens
- 缓存未命中: ¥1/百万tokens

**节省策略**:
- 集中批量处理相似问题
- 避免频繁重复相似查询
- 提供简洁但足够详细的上下文

### 提示词优化

```
以下是一个高效的提示词模板:

请帮我实现一个[功能描述]功能，使用[技术栈]。
具体要求:
1. [具体要求1]
2. [具体要求2]
3. [具体要求3]

示例输入: [输入示例]
期望输出: [输出示例]

请先解释实现思路，然后提供完整代码实现。
```

### 与版本控制结合使用

1. 使用[代码版本控制](/resources/ai-programming-tutorials/version-control-tutorial)记录重要更改
2. 在每次重大AI生成后创建一个版本点
3. 这样可以安全地探索不同实现方案而不担心丢失好的解决方案
