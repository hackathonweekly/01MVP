**3.6 部署与备案**

**部署网站的几个方案优缺点分析**

1. 国外域名 + 国外云服务器（适合小流量访问的网站）
1. 优点：流程非常简单
2. 缺点：国内访问不稳定，延迟高 + 有被微信屏蔽的风险，无法收付款，不能支持微信登录
2. ~~国外域名 + 国内服务器 or CDN（方案不成立：国内服务器 or CDN 必须备案）~~
3. ~~国内域名备案 + 国外服务器（方案不成立：备案网站必须部署在国内）~~
4. 国内域名备案 + 国外服务器 + 国内 CDN （适合中小流量，需要支付功能网站）
1. 优点：国内访问速度**一般**，能利用海外便利且便宜的云服务
2. 缺点：国外访问延迟较高（因为走了国内 CDN），配置略微麻烦
3. 折中方案 1 ：[双 CDN 方案](https://linux.do/t/topic/212918/14)，国内走国内 CND，海外走海外 CDN，配置麻烦。利用到了国内云提供的 DNS 分区域解析域名功能
4. 折中方案 2：完全选择国内云，使用他们的全球加速，但价格较高。
5. 国内域名备案 + 国内服务器（适合只要国内访问的网站）
1. 优点：国内访问速度快
2. 缺点：无法使用国外便利云服务；价格较高
6. ` `国内国外双域名+双部署方案，国内 xx.cn ，国外 xx.com （最佳方案，案例：飞书/canvas）
